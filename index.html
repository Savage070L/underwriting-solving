Цвет строки — статус договора
Красный — некорректные данные, исправить в базе Жёлтый — расхождения, нужна доп. проверка Зелёный — все данные корректны Синий — проверяется Серый — в очереди
Цвет ячейки — почему значение подсвечено
Красная — значение неверно. СС < ФОТ (страхователя или контрагента); СС страхователя < страховой премии; премия < 1 МЗП (85 000); при премии < 1 МЗП СС контрагента ≠ ФОТ; СП контрагента ≠ СС×тариф×ПК; Σ СС или Σ СП контрагентов ≠ итогу страхователя; ОКЭД (С) отсутствует у компании по stat.gov.kz; класс (С) не соответствует классификатору для своего ОКЭД; класс (К) отсутствует среди ОКЭД контрагента; тариф (К) не соответствует справочнику; невалидный БИН; скидка ПК (0,9) при компании моложе 3 лет; гос. доля ≠ реестру e-Qazyna.
Жёлтая — мягкое расхождение. Класс (С или К) ≥ 13 есть среди ОКЭД компании, но не максимальный из её видов деятельности. Расхождения среди классов 1–12 не выделяются (считаются нормой).
Печать рекомендации ДАиП

По каждому договору из реестра печатается «Рекомендация ДАиП» (один документ на договор, филиалы — внутри), все в один ZIP. Реестр берётся из вкладки «Проверка договоров». Проверка stat.gov.kz не требуется.

Реестр не загружен. Загрузите выгрузку договоров во вкладке — она появится здесь автоматически.
Проверка контрагента по БИН

Введите БИН/ИИН — приложение проверит компанию в реестрах (stat.gov.kz, гос. участие, аффилированные лица, юр. адрес, дата регистрации).
